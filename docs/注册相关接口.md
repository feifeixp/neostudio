# 用户认证接口文档（节选：registerByMobile / checkUserExist）

**Base URL**
- `/user/login/`

**通用说明**
- 请求方式均为 `POST`
- 请求体 `Content-Type: application/json`
- 返回结构使用 COLA：
  - `Response` / `SingleResponse<T>`

---

## 1. 用户注册（支持手机号/邮箱）- registerByMobile

### 基本信息
- **接口名**: 用户注册(支持手机号和邮箱)
- **URL**: `POST /user/login/register`
- **Controller 方法**: `LoginController.registerByMobile(...)`

### Header 参数
| 参数名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `channelCode` | string | 否 | 渠道码（从请求头获取） |

### Body 参数（MobileRegisterReq）
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `invitationCode` | string | 是 | 邀请码 |
| `mobile` | string | 是 | 联系方式（方法内使用 `req.getMobile()`，语义为手机号/邮箱，具体校验规则以 `MobileRegisterReq` 上的校验注解为准） |
| `verifyCode` | string | 是 | 验证码 |

### 请求示例
```json
{
  "invitationCode": "INV123456",
  "mobile": "13800138000",
  "verifyCode": "123456"
}
```

### 成功响应
- **返回类型**: `SingleResponse<UserLoginRes>`
- **data**: `UserLoginRes`（具体字段以 `UserLoginRes` 定义为准）

响应示例（结构示意）：
```json
{
  "success": true,
  "errCode": null,
  "errMessage": null,
  "data": {
    "token": "xxx"
  }
}
```

### 失败响应
- Controller 未显式构造失败响应（无 `buildFailure`），失败通常来自：
  - `@Valid` 参数校验失败
  - `loginService.registerByMobile(...)` 内部业务校验/异常

---

## 2. 检查用户是否存在（支持手机号/邮箱）- checkUserExist

### 基本信息
- **接口名**: 检查用户是否存在(支持手机号和邮箱)
- **URL**: `POST /user/login/check-exist`
- **Controller 方法**: `LoginController.checkUserExist(...)`

### Body 参数（CheckUserExistReq）
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `contact` | string | 是 | 联系方式（手机号或邮箱） |

### 请求校验（Controller 内置）
- **手机号**: `^1[3-9]\d{9}$`
- **邮箱**: `^[a-zA-Z0-9_-]+@[a-zA-Z0-9_-]+(\.[a-zA-Z0-9_-]+)+$`
- 若 `contact` 不匹配手机号或邮箱格式，直接返回失败。

### 请求示例
```json
{
  "contact": "test@example.com"
}
```

### 成功响应
- **返回类型**: `SingleResponse<CheckUserExistRes>`

从日志可推断 `CheckUserExistRes` 至少包含：
| 字段 | 类型 | 说明 |
|---|---|---|
| `exists` | boolean | 是否存在 |
| `contactType` | string | 联系方式类型（具体枚举/取值以服务实现为准） |

响应示例（结构示意）：
```json
{
  "success": true,
  "errCode": null,
  "errMessage": null,
  "data": {
    "exists": true,
    "contactType": "mobile"
  }
}
```

### 失败响应（明确）
当 `contact` 格式非法：
- **errCode**: `PARAM_ERROR`
- **errMessage**: `请输入有效的手机号或邮箱地址`

响应示例：
```json
{
  "success": false,
  "errCode": "PARAM_ERROR",
  "errMessage": "请输入有效的手机号或邮箱地址",
  "data": null
}
```
