## 1. 分页查询协作项目

### 接口信息

- **接口路径**: `/agent/project-collaboration/page-query-projects`
- **请求方法**: `POST`
- **接口描述**: 分页查询所有协作项目，支持按用户ID、会话ID和项目名称进行筛选，返回项目详细信息和成员列表

### 请求参数

**Content-Type**: `application/json`

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| sessionId | String | 否 | 会话ID，用于查询指定会话的协作项目 | "session123" |
| userId | String | 否 | 用户ID，用于筛选该用户参与的协作项目 | "user123" |
| projectName | String | 否 | 项目名称关键词，支持模糊搜索 | "测试项目" |
| pageNum | Integer | 否 | 页码，从1开始，默认为1 | 1 |
| pageSize | Integer | 否 | 每页大小，默认为20 | 20 |

### 请求示例

```json
{
  "sessionId": "session123",
  "userId": "user123",
  "projectName": "测试项目",
  "pageNum": 1,
  "pageSize": 20
}
```

### 响应参数

**返回格式**: `PageResponse<CollaborationProjectRes>`

| 参数名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 请求是否成功 |
| data | Array | 协作项目列表 |
| totalCount | Integer | 总记录数 |
| pageSize | Integer | 每页大小 |
| pageIndex | Integer | 当前页码 |
| totalPages | Integer | 总页数 |

**data 数组元素字段 (CollaborationProjectRes)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| sessionId | String | 会话ID |
| projectName | String | 项目名称（会话标题） |
| description | String | 项目描述 |
| creatorUserId | String | 项目创建者用户ID |
| creatorNickname | String | 项目创建者昵称 |
| projectType | Integer | 项目类型：0-私有项目，1-协作项目 |
| memberCount | Integer | 协作成员数量 |
| members | Array | 项目成员列表 |
| balanceLeftPoints | Integer | 项目剩余积分 |
| sessionType | Integer | 会话类型：1-普通模式，2-主角模式，3-漫剧模式，4-普通模式2，5-角色模式 |
| createTime | DateTime | 项目创建时间 |
| updateTime | DateTime | 项目更新时间 |

**members 数组元素字段 (ProjectMemberBriefRes)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| userId | String | 用户ID |
| nickname | String | 用户昵称 |
| avatar | String | 用户头像URL |

### 响应示例

```json
{
  "success": true,
  "data": [
    {
      "sessionId": "session123",
      "projectName": "测试项目",
      "description": "这是一个测试项目",
      "creatorUserId": "creator123",
      "creatorNickname": "创建者昵称",
      "projectType": 1,
      "memberCount": 3,
      "members": [
        {
          "userId": "user001",
          "nickname": "张三",
          "avatar": "https://example.com/avatar1.jpg"
        },
        {
          "userId": "user002",
          "nickname": "李四",
          "avatar": "https://example.com/avatar2.jpg"
        },
        {
          "userId": "user003",
          "nickname": "王五",
          "avatar": "https://example.com/avatar3.jpg"
        }
      ],
      "balanceLeftPoints": 750,
      "sessionType": 5,
      "createTime": "2026-01-27T10:00:00",
      "updateTime": "2026-01-27T12:00:00"
    }
  ],
  "totalCount": 10,
  "pageSize": 20,
  "pageIndex": 1,
  "totalPages": 1
}
```

### 功能特性

- ✅ 支持按会话ID查询指定会话的协作项目
- ✅ 支持按用户ID筛选该用户参与的所有协作项目
- ✅ 支持按项目名称进行模糊搜索
- ✅ 返回项目详细信息（名称、描述、创建者、成员数量等）
- ✅ 返回完整的项目成员列表（包含用户ID、昵称、头像）
- ✅ 返回项目积分信息和会话类型
- ✅ 按项目创建时间倒序排列

## 2. 查询项目积分操作记录

### 接口信息

- **接口路径**: `/agent/project-collaboration/points-history`
- **请求方法**: `GET`
- **接口描述**: 根据会话ID分页查询项目的积分操作记录，支持按类型筛选

### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| sessionId | String | 是 | 会话ID | "session123" |
| pageNum | Integer | 否 | 页码，从1开始，默认为1 | 1 |
| pageSize | Integer | 否 | 每页数量，默认为10 | 10 |
| queryType | Integer | 否 | 查询类型：1-获得，2-消耗，不传查询全部 | 1 |

### 请求示例

```
GET /agent/project-collaboration/points-history?sessionId=session123&pageNum=1&pageSize=10&queryType=2
```

### 响应参数

**返回格式**: `PageResponse<PointsHistoryRes>`

| 参数名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 请求是否成功 |
| data | Array | 积分操作记录列表 |
| totalCount | Integer | 总记录数 |
| pageSize | Integer | 每页大小 |
| pageIndex | Integer | 当前页码 |
| totalPages | Integer | 总页数 |

**data 数组元素字段 (PointsHistoryRes)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 记录ID |
| description | String | 变更描述（国际化） |
| pointsChange | Integer | 积分变更值（正数表示增加，负数表示减少） |
| balanceAfterChange | Integer | 变更后的余额 |
| timestamp | DateTime | 变更时间 |
| relatedBusinessId | String | 关联业务ID |

### 响应示例

```json
{
  "success": true,
  "data": [
    {
      "id": 12345,
      "description": "会话消耗",
      "pointsChange": -100,
      "balanceAfterChange": 900,
      "timestamp": "2026-01-27T14:00:00",
      "relatedBusinessId": "session123"
    },
    {
      "id": 12344,
      "description": "系统赠送",
      "pointsChange": 50,
      "balanceAfterChange": 1000,
      "timestamp": "2026-01-27T13:00:00",
      "relatedBusinessId": "gift001"
    }
  ],
  "totalCount": 50,
  "pageSize": 10,
  "pageIndex": 1,
  "totalPages": 5
}
```

### 功能特性

- ✅ 根据会话ID查询项目积分操作记录
- ✅ 支持按类型筛选（获得/消耗）
- ✅ 只查询最近一个月的数据
- ✅ 按创建时间倒序排列
- ✅ 国际化的交易类型描述

## 3. 分页查询用户列表

### 接口信息

- **接口路径**: `/user/list`
- **请求方法**: `GET`
- **接口描述**: 分页查询所有用户列表，支持按用户ID和昵称进行筛选

### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| pageNum | Integer | 否 | 页码，从1开始，默认为1 | 1 |
| pageSize | Integer | 否 | 每页数量，默认为10 | 10 |
| userId | String | 否 | 用户ID，精确匹配 | "user123" |
| nickname | String | 否 | 用户昵称，支持模糊查询 | "张三" |

### 请求示例

```
GET /user/list?pageNum=1&pageSize=10&nickname=张三
```

### 响应参数

**返回格式**: `PageResponse<UserDetailRes>`

| 参数名 | 类型 | 说明 |
|--------|------|------|
| success | Boolean | 请求是否成功 |
| data | Array | 用户详细信息列表 |
| totalCount | Integer | 总记录数 |
| pageSize | Integer | 每页大小 |
| pageIndex | Integer | 当前页码 |
| totalPages | Integer | 总页数 |

**data 数组元素字段 (UserDetailRes)**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| userId | String | 用户ID |
| nickname | String | 用户昵称 |
| email | String | 用户邮箱 |
| mobile | String | 用户手机号（脱敏），格式：138****8000 |
| avatar | String | 用户头像URL |
| status | Integer | 用户状态：0-未激活，1-正常，2-禁用 |
| region | String | 用户地区 |
| job | String | 用户职业 |
| age | String | 用户年龄 |

| activationTime | DateTime | 账户激活时间 |

| updateTime | DateTime | 更新时间 |

## 通用说明

### 分页响应格式

所有接口均使用 COLA 框架的 `PageResponse` 标准分页格式：

```json
{
  "success": true,
  "data": [...],
  "totalCount": 100,
  "pageSize": 10,
  "pageIndex": 1,
  "totalPages": 10
}
```

### 错误响应

当请求失败时，返回格式如下：

```json
{
  "success": false,
  "errCode": "ERROR_CODE",
  "errMessage": "错误描述信息"
}
```

### 注意事项

1. 所有时间字段均为 ISO 8601 格式：`yyyy-MM-ddTHH:mm:ss`
2. 分页参数 `pageNum` 和 `pageSize` 均有默认值，可不传
3. 查询类型参数为可选，不传则查询全部数据
4. 手机号等敏感信息会自动脱敏处理
5. 积分操作记录只查询最近一个月的数据
