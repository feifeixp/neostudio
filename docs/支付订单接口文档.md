# 支付订单接口文档

## 1. 文档说明

本文档整理以下 3 个支付相关接口：

- `createOrder`：创建支付订单
- `queryPayStatus`：查询支付状态
- `closeOrder`：关闭订单

接口统一归属控制器：`PayController`

接口基础路径：`/agent/pay`

说明：

- 业务代码中会从 `UserContext` 获取当前用户信息，因此这些接口应在登录态下调用。
- 返回结构使用 COLA DTO 风格，成功时通常包含 `success`、`data`、`errCode`、`errMessage` 等字段。

---

## 2. 创建支付订单

### 接口信息

- **接口名称**: `createOrder`
- **接口路径**: `/agent/pay/order/create`
- **请求方式**: `POST`
- **请求类型**: `application/json`
- **接口描述**: 创建支付订单并返回支付所需参数

### 请求参数

#### Body (JSON)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| subject | String | 是 | 商品标题 | `会员月卡` |
| body | String | 否 | 商品描述 | `会员服务1个月` |
| amount | Double | 是 | 订单金额，单位：元，必须大于等于 0 | `9.9` |
| payType | Integer | 是 | 支付方式，当前仅支持微信支付 | `1` |

### 参数校验规则

- `subject` 不能为空
- `amount` 不能为空，且最小值为 `0`
- `payType` 不能为空
- 当前支付方式枚举：
  - `1`：微信支付

### 请求示例

```json
{
  "subject": "会员月卡",
  "body": "会员服务1个月",
  "amount": 9.9,
  "payType": 1
}
```

### 响应字段说明

响应体类型：`SingleResponse<PayOrderRes>`

#### data 字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| orderNo | String | 订单编号 |
| subject | String | 商品标题 |
| amount | Double | 订单金额，单位：元 |
| payType | Integer | 支付方式 |
| prepayId | String | 微信预支付 ID，部分支付场景返回 |
| codeUrl | String | 微信 Native 支付二维码链接 |
| mwebUrl | String | 微信 H5 支付链接，部分支付场景返回 |
| wxPayParams | Object | 微信小程序支付参数，部分支付场景返回 |
| status | Integer | 订单状态：`0-未支付`，`1-支付中`，`2-已支付`，`3-已取消`，`4-已退款` |

#### wxPayParams 字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| appId | String | 小程序 `appId` |
| timeStamp | String | 时间戳 |
| nonceStr | String | 随机字符串 |
| packageValue | String | 订单扩展包，一般格式为 `prepay_id=xxx` |
| signType | String | 签名方式 |
| paySign | String | 支付签名 |

### 成功响应示例

```json
{
  "success": true,
  "data": {
    "orderNo": "20260330111900123456",
    "subject": "会员月卡",
    "amount": 9.9,
    "payType": 1,
    "prepayId": null,
    "codeUrl": "weixin://wxpay/bizpayurl?pr=xxxxxx",
    "mwebUrl": null,
    "wxPayParams": null,
    "status": 0
  },
  "errCode": null,
  "errMessage": null
}
```

### 失败场景

| 场景 | 说明 |
|------|------|
| 参数校验失败 | 如 `subject` 为空、`amount` 为空或小于 0、`payType` 为空 |
| 下单失败 | 微信下单失败时抛出业务异常 |

已知业务错误码：

| 错误码 | 含义 |
|--------|------|
| `4133` | 微信支付下单失败 |

## 3. 查询支付状态

### 接口信息

- **接口名称**: `queryPayStatus`
- **接口路径**: `/agent/pay/order/status`
- **请求方式**: `GET`
- **接口描述**: 轮询查询订单支付状态，返回轻量级订单状态信息

### 请求参数

#### Query 参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| orderNo | String | 是 | 订单编号 | `20260330111900123456` |

### 请求示例

```http
GET /agent/pay/order/status?orderNo=20260330111900123456
```

### 响应字段说明

响应体类型：`SingleResponse<PayOrderStatusRes>`

#### data 字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| orderNo | String | 订单编号 |
| amount | Double | 订单金额，单位：元 |
| subject | String | 商品标题 |
| status | Integer | 订单状态：`0-未支付`，`1-支付中`，`2-已支付`，`3-已取消`，`4-已退款` |
| statusDesc | String | 状态描述：`未支付`、`支付中`、`已支付`、`已取消`、`已退款` |
| createTime | Date | 创建时间 |
| payTime | Date | 支付时间，支付成功时有值 |

### 成功响应示例

```json
{
  "success": true,
  "data": {
    "orderNo": "20260330111900123456",
    "amount": 9.9,
    "subject": "会员月卡",
    "status": 2,
    "statusDesc": "已支付",
    "createTime": "2026-03-30 11:19:00",
    "payTime": "2026-03-30 11:20:15"
  },
  "errCode": null,
  "errMessage": null
}
```

### 说明

- 该接口内部通过 `payService.queryOrder(orderNo)` 查询本地订单记录。
- 返回的 `amount` 会从数据库中的“分”转换为“元”后输出。
- 该接口适合前端轮询支付结果。

### 失败场景

| 场景 | 说明 |
|------|------|
| `orderNo` 为空 | 返回订单号不能为空错误 |
| 订单不存在或查询异常 | 依赖底层查询逻辑返回异常 |

已知业务错误码：

| 错误码 | 含义 |
|--------|------|
| `4134` | 订单号不能为空 |

## 4. 关闭订单

### 接口信息

- **接口名称**: `closeOrder`
- **接口路径**: `/agent/pay/order/close`
- **请求方式**: `POST`
- **接口描述**: 关闭指定订单

### 请求参数

#### Query/Form 参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| orderNo | String | 是 | 订单编号 | `20260330111900123456` |

### 请求示例

```http
POST /agent/pay/order/close?orderNo=20260330111900123456
```

### 响应字段说明

响应体类型：`Response`

### 成功响应示例

```json
{
  "success": true,
  "errCode": null,
  "errMessage": null
}
```

### 业务规则

根据当前实现，关闭订单前会进行以下校验：

- `orderNo` 不能为空
- 订单必须存在
- **当前代码要求订单状态必须为 `1-支付中` 才允许关闭**

关闭成功后会执行：

- 调用微信支付关闭订单接口
- 将本地订单状态更新为 `3-已取消`
- 同步关联订单状态为已取消
- 若存在交易记录，则将交易状态更新为失败

### 失败场景

| 场景 | 说明 |
|------|------|
| `orderNo` 为空 | 返回订单号不能为空错误 |
| 订单不存在 | 返回订单不存在错误 |
| 订单状态不允许关闭 | 当前仅允许 `支付中` 状态关闭 |
| 微信侧关闭失败 | 返回关闭订单失败错误 |

已知业务错误码：

| 错误码 | 含义 |
|--------|------|
| `4134` | 订单号不能为空 |
| `4135` | 订单不存在 |
| `4136` | 只有支付中的订单才能关闭 |
| `4137` | 关闭订单失败 |

---

## 5. 订单状态枚举

| 状态值 | 状态说明 |
|--------|----------|
| `0` | 未支付 |
| `1` | 支付中 |
| `2` | 已支付 |
| `3` | 已取消 |
| `4` | 已退款 |

---

## 6. 通用说明

### 通用响应格式

#### 1）带 data 的响应

```json
{
  "success": true,
  "data": {},
  "errCode": null,
  "errMessage": null
}
```

#### 2）无 data 的响应

```json
{
  "success": true,
  "errCode": null,
  "errMessage": null
}
```

### 注意事项

1. 创建订单接口中的 `amount` 入参单位为“元”，系统内部会转换为“分”存储。
2. 查询支付状态接口返回的 `amount` 单位为“元”。
3. 当前 `payType` 仅支持 `1-微信支付`。
4. `queryPayStatus` 查询的是本地订单状态；如果需要主动向微信同步，可使用项目中的 `/agent/pay/order/sync` 接口，但该接口不在本文档整理范围内。
5. `closeOrder` 当前实际实现要求订单处于 `支付中` 状态才允许关闭，调用前建议先通过状态查询接口确认。
